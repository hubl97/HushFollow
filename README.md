# HushFollow

🔍 **悄悄关注，不留痕迹**

一个轻量级的微博博主管理工具。帮你收藏微博博主主页链接，无需在平台上直接关注，所有数据保存在你的本地文件中。

## 一句话介绍

把 `hushfollow.html` 双击打开，粘贴微博主页链接即可添加博主。数据自动保存为 Markdown 文件，可上传 GitHub、网盘随意同步。零安装，零依赖，零网络请求。

## 使用方式

**只需一个文件，双击即用：**

1. 下载 [hushfollow.html](hushfollow.html) 到本地
2. 双击用浏览器打开
3. 粘贴微博博主主页链接，添加即可

**日常操作：**

- ➕ **添加博主** — 粘贴微博主页链接，可选填昵称
- 🔎 **搜索博主** — 输入关键词实时筛选
- ✏️ **编辑昵称** — 点击编辑按钮，失焦自动保存
- 🗑️ **删除博主** — 确认后移除
- 📂 **打开文件** — 加载已有的 `.md` 数据文件
- 📥 **导入合并** — 将另一个 `.md` 文件的内容合并到当前列表（自动去重）
- 💾 **保存 / 另存为** — 导出为 Markdown 文件

## 数据格式

所有数据以纯 Markdown 列表存储，人类可读、可手动编辑：

```markdown
# HushFollow

- [博主昵称A](https://weibo.com/u/1234567890)
- [博主昵称B](https://weibo.com/u/0987654321)
```

你可以直接把这些文件放到 GitHub 仓库里，在 GitHub 上渲染出来就是可点击的博主列表。

## 常见问题

**Q: 我的数据安全吗？**
A: 非常安全。应用不连接任何服务器，不使用 Cookie 或分析工具。所有数据保存在你的浏览器 localStorage 和本地 `.md` 文件中。

**Q: 关闭浏览器后数据会丢失吗？**
A: 不会。数据会自动保存到浏览器，下次打开自动恢复。

**Q: 如何备份和同步？**
A: 将生成的 `.md` 文件上传到 GitHub、网盘或发送到邮箱。在其他设备上下载后用"打开文件"加载即可。

**Q: 换电脑怎么办？**
A: 复制 `hushfollow.html` 和你的 `.md` 文件到新电脑，打开 HTML 后选择"打开文件"加载数据。

**Q: 刷新页面后自动保存不工作了？**
A: 浏览器安全限制导致文件句柄在刷新后失效。点击"保存"按钮重新选择文件即可恢复自动保存。

**Q: 支持哪些微博链接格式？**
A: 支持 `weibo.com/u/数字ID` 和 `weibo.com/自定义域名` 格式。
