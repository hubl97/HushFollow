# HushFollow

🔍 悄悄关注，不留痕迹

一个轻量级的单文件 HTML 应用，帮助你管理微博博主列表，无需在平台上直接关注。

## 功能特点

- ✨ **单文件应用**：只有一个 HTML 文件，双击即用，无需安装
- 💾 **自动保存**：添加/编辑/删除后自动保存，无需手动点击
- 🧠 **记住状态**：关闭页面后自动恢复上次数据，无需重复导入
- 🔒 **数据自主**：所有数据存储在你的本地 Markdown 文件中
- ☁️ **自由同步**：可将文件上传到 GitHub、网盘等任意云服务
- 🌐 **跨平台**：Windows、Mac、Linux、手机都能用
- 📱 **响应式设计**：适配桌面和移动设备
- 🚀 **零依赖**：无需 Node.js、Python 或任何运行环境

## 快速开始

1. **下载** `hushfollow.html` 文件
2. **双击打开**（浏览器会自动打开）
3. **首次使用**：点击 **"快速开始"** 生成示例文件
4. **日常使用**：直接添加博主，自动保存

## 使用指南

### 添加博主

1. 在"微博链接"输入框粘贴博主主页地址，例如：
   ```
   https://weibo.com/u/6351705477
   ```
2. （可选）在"昵称"输入框输入自定义名称
3. 点击 **"添加"** 按钮或按 Enter 键
4. 数据**自动保存**到 localStorage 和文件

> 💡 如果不输入昵称，系统会自动使用微博用户 ID 作为临时昵称

### 管理博主

- **访问**：点击博主卡片上的 **"访问"** 按钮，在新标签页打开微博主页
- **编辑**：点击 **"编辑"** 按钮修改昵称，失焦后自动保存
- **删除**：点击 **"删除"** 按钮，确认后移除并自动保存
- **搜索**：在搜索框输入关键词实时筛选

### 文件操作

| 按钮 | 功能 | 说明 |
|------|------|------|
| 📂 **打开文件** | 替换当前列表 | 打开新的 `.md` 文件，**替换**当前所有博主 |
| 📥 **导入合并** | 合并到当前列表 | 打开 `.md` 文件，**合并**到现有列表（自动去重） |
| 💾 **保存** | 保存到当前文件 | 直接覆盖原文件（Chrome/Edge） |
| 📤 **另存为** | 导出为新文件 | 下载新的 `.md` 文件 |
| 📁 **文件信息** | 查看文件详情 | 显示文件名、博主数量、修改时间 |

### 自动保存机制

**双重存储策略：**

1. **localStorage（即时）**：每次操作后立即保存到浏览器，关闭页面不丢失
2. **文件系统（延迟1秒）**：操作后1秒自动保存到 `.md` 文件

**触发条件：**
- 添加博主 → 自动保存
- 编辑昵称 → 自动保存
- 删除博主 → 自动保存
- 导入合并 → 自动保存

**状态显示：**
- 🟢 **已保存** - 数据已同步
- 🟡 **未保存** - 等待自动保存（1秒后）
- 🔵 **保存中...** - 正在写入文件

### 首次使用 vs 后续使用

**首次使用：**
- 显示欢迎界面，只有"快速开始"按钮
- 点击后下载示例文件并加载数据
- 后续不再显示欢迎界面

**后续使用：**
- 自动恢复上次的数据（从 localStorage）
- 无需重复导入文件
- 直接继续编辑

### 数据同步

1. 将 `.md` 文件上传到你的 GitHub 仓库或网盘
2. 在其他设备上下载该文件
3. 使用 HushFollow 打开即可继续编辑

## 常见问题

**Q: 我的数据存储在哪里？**
A: 数据存储在两个地方：
   - **临时存储**：浏览器 localStorage（自动保存，关闭不丢失）
   - **永久存储**：你自己保存的 `.md` 文件（可上传到云盘）

**Q: 我把 .md 文件移动位置了，打开应用为什么还显示旧数据？**
A: 由于浏览器安全限制，网页应用无法验证本地文件是否被移动。启动时会显示"发现缓存数据"界面，你可以选择：
   - **📂 继续使用缓存数据** - 加载浏览器缓存的数据
   - **📂 打开文件（重新锚定）** - 选择文件的新位置
   - **🗑️ 清空并重新开始** - 清除所有缓存数据

**Q: 关闭页面后数据会丢失吗？**
A: 不会。数据会自动保存到浏览器的 localStorage，下次打开自动恢复。

**Q: 如何备份数据？**
A: 将 `.md` 文件上传到 GitHub 仓库、网盘或发送到邮箱即可备份。

**Q: 换电脑后如何使用？**
A: 在新电脑上下载 `hushfollow.html` 和你的 `.md` 文件，打开即可。

**Q: 自动保存为什么不工作？**
A: 首次使用需要先手动点击"保存"按钮一次，设置文件位置后自动保存才会生效。

**Q: 支持哪些微博链接格式？**
A: 支持 `https://weibo.com/u/{数字ID}` 和 `https://weibo.com/{自定义域名}` 格式。

**Q: 文件信息里的"存储位置"为什么显示不完整？**
A: 浏览器安全限制，网页应用无法获取本地文件的完整路径。建议使用快捷键 `Ctrl+J` 打开下载列表，右键文件选择"在文件夹中显示"。
