# HushFollow

Follow with grace, leave no trace.

微博取消了悄悄关注功能。HushFollow 是一个离线工具，帮你收藏博主主页链接，不产生任何关注记录，不暴露任何兴趣痕迹。

一个 HTML 文件，双击打开。没有安装，没有注册，没有网络请求。

---

## 为什么做这个

微博取消了悄悄关注。

**但关注应该是你自己的事。**

你想看谁的内容，不想让谁知道——这不叫隐瞒，这叫边界。HushFollow 把这种自由还给你。

---

## 快速开始

1. 下载 [hushfollow.html](hushfollow.html)
2. 双击用浏览器打开
3. 粘贴微博主页链接，添加即可

就这么简单。

## 你能做什么

- **添加博主** — 粘贴链接，可选填昵称
- **搜索** — 输入关键词，实时筛选
- **编辑昵称** — 点击编辑，失焦自动保存
- **拖拽排序** — 按住卡片拖动，按你的习惯排列
- **删除** — 确认后移除
- **打开文件** — 加载已有的数据文件
- **导入合并** — 合并另一个文件的内容，自动去重
- **保存 / 另存为** — 导出为 Markdown 文件
- **暗黑模式** — 自动跟随系统，也可手动切换

## 隐私

- 不连接任何服务器
- 不发一条网络请求
- 不使用 Cookie，不做追踪
- 不需要账号，不需要登录
- 所有数据只存在你的设备上

你的关注列表是你的隐私。HushFollow 尊重这一点。

## 数据格式

数据以纯 Markdown 存储，人类可读、可编辑、可版本控制：

```markdown
# HushFollow
version: 1

- [博主昵称A](https://weibo.com/u/1234567890)
- [博主昵称B](https://weibo.com/u/0987654321)
```

你可以把这个文件放到 GitHub 仓库里，渲染出来就是可点击的博主列表。用 Git 做版本管理，用网盘做同步——数据是你自己的，想怎么用就怎么用。

## 常见问题

**数据安全吗？**
绝对安全。没有任何服务器，没有 Cookie，没有追踪。数据只在你的浏览器和本地文件中。

**关闭浏览器会丢数据吗？**
不会。数据保存在浏览器 localStorage 和本地文件中，下次打开自动恢复。

**如何备份？**
导出 `.md` 文件，放到 GitHub、网盘或发到邮箱。新设备上打开 HTML 文件，用"打开文件"加载即可。

**支持哪些链接格式？**
`weibo.com/u/数字ID` 和 `weibo.com/自定义域名`。

## 技术实现

纯 HTML + CSS + JS，单文件，零依赖。

localStorage 做状态持久化，IndexedDB 持久化文件句柄，File System Access API 做文件读写（含降级方案）。
